# AGENTS.md Generation

The `agents-md` command generates an `AGENTS.md` file that teaches AI coding agents (Claude Code, Cursor, Copilot, etc.) how to use `grafana-assistant` inside a project.

## Why

AI agents read `AGENTS.md` at the start of a session to learn what tools are available. Without it, an agent has no way to know that `grafana-assistant` exists, what instances are configured, or how to query Grafana.

Running `agents-md` once gives every agent working with your repo immediate access to Grafana observability data.

## Quick Start

```bash
grafana-assistant agents-md
```

This runs interactively, asking which configured instances, datasource names, and default namespace to include. The output is an `AGENTS.md` file in the current directory.

## Interactive Prompts

When run without `--non-interactive`, the command asks three questions:

1. **Instances** — picks from your configured instances (`grafana-assistant config list`) and embeds their names and URLs.
2. **Datasource names** — comma-separated names (e.g. `grafana-cloud-metrics, grafana-cloud-logs`). Used in example queries so the agent targets the right datasource.
3. **Namespace** — the default Kubernetes namespace (e.g. `backend-prod`). Used in example queries.

Press Enter to skip any prompt. Skipped values use generic placeholders.

## Merge Behavior

| Scenario | Behavior |
|----------|----------|
| No `AGENTS.md` exists | Creates a new file |
| `AGENTS.md` exists, no `--force` | Appends or replaces the `grafana-assistant` section (between `<!-- BEGIN/END grafana-assistant -->` markers), preserving everything else |
| `AGENTS.md` exists, `--force` | Overwrites the entire file |

This makes it safe to re-run `agents-md` after upgrading without losing custom content in the file.

## Flags

| Flag | Description | Default |
|------|-------------|---------|
| `-o, --output` | Output filename | `AGENTS.md` |
| `-f, --force` | Overwrite entire file | `false` |
| `--dry-run` | Print to stdout instead of writing | `false` |
| `--non-interactive` | Skip prompts, use generic placeholders | `false` |

## Examples

```bash
# Interactive — picks instances, datasources, namespace from prompts
grafana-assistant agents-md

# Non-interactive for CI
grafana-assistant agents-md --non-interactive

# Preview without writing
grafana-assistant agents-md --dry-run

# Generate into a specific project directory
grafana-assistant agents-md ~/projects/my-service

# Overwrite an existing file completely
grafana-assistant agents-md --force
```

## What Gets Generated

The generated section includes:

- **Setup** — configured instance names and URLs (or generic setup steps)
- **Querying** — `prompt` command usage with `--json` and `--continue`
- **Example queries** — with the project's actual datasource and namespace
- **Common mistakes** — gotchas that cause agents to produce bad queries
- **Local tool access** — `add-project` and `tunnel connect` commands

The content is designed to be concise. Following the principle from [Claude Code best practices](https://code.claude.com/docs/en/best-practices#write-an-effective-claude-md): only include things the agent can't figure out by reading code.

## Customizing

The generated file ends with:

```
*Generated by `grafana-assistant agents-md`. Edit freely.*
```

You can edit anything inside the `<!-- BEGIN/END grafana-assistant -->` markers, but re-running `agents-md` will replace that section. Add custom content outside the markers to preserve it across regenerations.
